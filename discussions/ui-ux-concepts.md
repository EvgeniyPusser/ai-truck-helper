# UI/UX Концепции для Holy Move

## Философия дизайна

### Для клиентов (B2C)
- **Простота:** Минимум шагов, понятные иконки
- **Скорость:** Быстрый поиск и заказ
- **Доверие:** Рейтинги, отзывы, верификация
- **Мобильность:** Оптимизация для мобильных устройств

### Для провайдеров (B2B)
- **Эффективность:** Быстрый доступ к важной информации
- **Детализация:** Статистика, аналитика, графики
- **Управление:** Инструменты для контроля бизнеса
- **Интеграция:** Уведомления, календари

## Ключевые экраны

### 1. Landing / Entry Point
```
[ Holy Move Logo ]
     ↓
[ Я ПЕРЕЕЗЖАЮ ] [ Я ПРЕДОСТАВЛЯЮ УСЛУГИ ]
     ↓
Выбор типа: 🏠 Клиент 🚚 Водитель 💪 Грузчик 🏢 Компания 🤝 Агент
```

### 2. Клиентский путь
```
Поиск → Результаты → Детали заказа → Оформление → Отслеживание → Отзыв
```

### 3. Провайдерский путь
```
Дашборд → Заказы → Выполнение → Получение оплаты → Статистика
```

## Дизайн-система

### Цветовая палитра
- **Основной:** #2563EB (синий доверия)
- **Акцент:** #10B981 (зеленый успеха)
- **Внимание:** #F59E0B (оранжевый)
- **Ошибка:** #EF4444 (красный)
- **Нейтральный:** #6B7280 (серый)

### Типографика
- **Заголовки:** Inter, 600-700 weight
- **Текст:** Inter, 400 weight
- **Моноширинный:** JetBrains Mono (для кода/ID)

### Иконки
- **Навигация:** Простые, узнаваемые
- **Статусы:** Цветовые индикаторы
- **Действия:** Контекстные иконки

## Компоненты

### 1. Поисковая форма
```jsx
<SearchForm>
  <Input placeholder="Откуда (ZIP или адрес)" />
  <Input placeholder="Куда (ZIP или адрес)" />
  <Select>Тип услуги</Select>
  <DatePicker>Когда</DatePicker>
  <Button>Найти исполнителей</Button>
</SearchForm>
```

### 2. Карточка исполнителя
```jsx
<ProviderCard>
  <Avatar src={provider.avatar} />
  <Info>
    <Name>{provider.name}</Name>
    <Rating>⭐ {provider.rating}</Rating>
    <Price>${provider.price}/час</Price>
  </Info>
  <Actions>
    <Button variant="outline">Просмотр</Button>
    <Button>Заказать</Button>
  </Actions>
</ProviderCard>
```

### 3. Дашборд провайдера
```jsx
<Dashboard>
  <StatsGrid>
    <StatCard title="Активные заказы" value="5" />
    <StatCard title="Заработано сегодня" value="$450" />
    <StatCard title="Рейтинг" value="4.8⭐" />
  </StatsGrid>
  <QuickActions>
    <Button>Проверить новые заказы</Button>
    <Button>Обновить статус</Button>
  </QuickActions>
</Dashboard>
```

## Адаптивный дизайн

### Мобильные (< 768px)
- **Навигация:** Bottom navigation bar
- **Карты:** Вертикальный стек
- **Формы:** Полноэкранные модалки
- **Таблицы:** Карточный вид

### Планшеты (768px - 1024px)
- **Навигация:** Collapsible sidebar
- **Карты:** 2-колоночная сетка
- **Формы:** Встроенные в страницу

### Десктоп (> 1024px)
- **Навигация:** Fixed sidebar
- **Карты:** 3-4 колонки
- **Формы:** Многоколоночные

## Микровзаимодействия

### Анимации
- **Переходы:** 200ms ease-out
- **Загрузка:** Skeleton screens
- **Успех:** Green flash + checkmark
- **Ошибка:** Red shake + message

### Уведомления
- **Push:** Новые заказы, сообщения
- **In-app:** Статусы заказов, обновления
- **Email:** Подтверждения, отчеты

### Голосовой интерфейс (будущее)
- **Поиск голосом:** "Найти грузчиков в 90210"
- **Управление голосом:** "Начать заказ", "Статус заказа"

## Доступность

### WCAG 2.1 AA
- **Контрастность:** 4.5:1 минимум
- **Навигация:** Tab порядок понятный
- **Экранный читатель:** ARIA labels
- **Клавиатура:** Все функции доступны

### Языковая поддержка
- **Основной:** English
- **Дополнительные:** Español, 中文, Русский
- **Форматы:** Локализация дат, валют, адресов

## Производительность

### Метрики
- **FCP:** < 1.5s
- **LCP:** < 2.5s
- **FID:** < 100ms
- **CLS:** < 0.1

### Оптимизации
- **Images:** WebP, lazy loading
- **Code:** Tree shaking, code splitting
- **Cache:** Service worker, CDN
- **Bundle:** < 1MB gzipped
